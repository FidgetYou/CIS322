Let's take a ReST

lost_key


This is the only lost call that does not authenticate the requester and uses plain text input and output.


Request

The request does not provide values for any of the arguments though data and the signature fields must be present in the request.


Response

result 'OK' or 'FAIL'


key The LOST public key


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“result“ : “OK“ ,
“key“ : “bksaoudu...aolchecksya“

activate_user


Reactivates LOST access for a user or generates a new user account if needed.


Request

username OSNAP username to activate.


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“username“: “carter“
}

Response

result 'OK,' 'NEW,' or 'FAIL'


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“result“ : “OK“ ,
}

suspend_user


Revokes access for a user. If the user does not exist or access has already been revoked, this call has no effect.


Request

username OSNAP username to revoke.


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“username“: “carter“
}

Response

result 'OK'


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“result“ : “OK“ ,

list_products


Requests a listing of all products in LOST based on filter criteria. Additional filters may be provided and additional data may be returned. The requests of the minimum form documented in the example must be accepted and the produced result must have at least the structure and fields of example provided. Asset information may not be provided by this call.


Request

vendor Case insensitive string to match against vendor name.


description Case insensitive string to match against product description.


compartments Security tags for teh asset as a json array, all tage must be matched for compartment and level.


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“ vendor “: “ “,
“ description “: “ notepad “,
“ compartments “: []
}

Response

listing a JSON list containing the product information entries.


Example:


{
“ timestamp “: “2017-02-02 06:15:13“,
“result“ : “OK“ ,
“key“ : “bksaoudu...aolchecksya“
